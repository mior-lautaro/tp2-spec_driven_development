## Descripción del proyecto

Este repositorio contiene el desarrollo del Trabajo Práctico N.º 2 de Ingeniería de Software III.

El objetivo del trabajo es especificar los requerimientos de un sistema web para la organización y gestión de eventos académicos utilizando la metodología **Spec-Driven Development (SDD)**.

La documentación se encuentra organizada en especificaciones funcionales (Specs) y documentos generales del proyecto, que servirán como base para una futura implementación del sistema.

## Especificaciones desarrolladas

### Spec 1 - Gestión de Eventos

Permite la administración completa de eventos académicos, incluyendo su creación, modificación, publicación y consulta.

### Spec 2 - Inscripción de Participantes

Permite a los usuarios registrarse e inscribirse en eventos, respetando las reglas de negocio definidas para cupos y fechas de inscripción.

## Tecnologías previstas

Aunque este trabajo se centra en la especificación de requerimientos, el sistema está pensado para desarrollarse como una aplicación web utilizando una arquitectura cliente-servidor.

Tecnologías previstas:

- Frontend Web
- API REST
- Base de datos relacional
- Git + GitHub para control de versiones
